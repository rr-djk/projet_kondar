## Key Interactions to Verify

### Pilier acoustique
- Lancer main_pi.py → serveur WebSocket port 8765 actif
- Forcer l'outil (fraiseuse bois / perceuse) → alerte warn en < 3s
- Forcer davantage → alerte critical → overlay rouge + pause simulateur
- Touche Espace → reprise simulateur
- Clic MUTE 30s → alerte suspendue → compteur faux positifs incrémenté
- Déconnecter le Pi → badge "ACOUSTIC OFFLINE" visible dans l'UI
- Reconnecter le Pi → badge disparaît, reconnect backoff [1,2,5]s

### Pilier visuel
- Lancer main_laptop.py demo.nc → chemin G-code visible à l'écran
- Poser un objet orange dans le champ webcam → chemin rouge + chemin vert
- Cliquer CONFIRMER → point suit le chemin vert
- Double-cliquer CONFIRMER → comportement identique (idempotent)
- Retirer l'objet → retour au chemin original proposé
- Objet bloque TOUT les chemins → overlay "CHEMIN IMPOSSIBLE"
- Simulateur visuel sans Pi (via mock_server.py) → fonctionnel

### Baseline acoustique
- Clic BASELINE → badge "EN ATTENTE BASELINE" → barre de progression 10s → "Baseline active"
- Clic BASELINE pendant RUNNING_NORMAL → non-bloqué
- Clic BASELINE pendant alerte active → BLOQUÉ (désactivé)

## Edge Cases

### Réseau
- Pi non joignable au démarrage → badge ACOUSTIC OFFLINE immédiat (pas de timeout > 5s)
- Pi reboot pendant la démo → reconnect automatique, pas de crash

### UI
- Double-clic CONFIRMER → idempotent (pas de double-transition FSM)
- Clic BASELINE puis immédiatement obstacle détecté → pas de race condition
- Queue WebSocket pleine (> 50 events) → drop silencieux, pas de crash

### Hardware
- Camera index invalide → badge "CAMERA OFFLINE" affiché
- Lumières variables → ArUco fallback (throttlé toutes les 100ms)
- Fichier .nc vide → comportement défini (gcode_parser.py)

### Fonctions pures
- A* timeout 150ms → même comportement que "aucun chemin" (path_blocked=True)
- A* throttle 200ms → pas de recalcul entre 0 et 200ms après dernier replan
- ArUco throttle 100ms → ArUco ne tourne pas à chaque frame

## Critical Paths

1. **Démo 30s standard (reproduire 3x consécutives sans crash) :**
   Lancer → point bleu suit G-code → poser objet orange → chemin rouge/vert → cliquer CONFIRMER → point suit vert → retirer objet → retour original

2. **Alerte acoustique :**
   Machine normale → forcer outil → warn jaune en < 3s → forcer plus → critical rouge → pause 3s → Espace pour reprendre

3. **Connexion WebSocket Pi→Laptop :**
   main_pi.py up → main_laptop.py détecte → badge vert → event acoustique reçu → overlay correct

4. **Baseline flow :**
   App démarre → BASELINE désactivé si FSM != IDLE → clic BASELINE → 10s → Baseline active → alertes activées
