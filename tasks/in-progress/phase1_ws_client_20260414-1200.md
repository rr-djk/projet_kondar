# T-1B.4 — ws_client.py (AcousticLink)

## Description
Implémenter `AcousticLink` — client WebSocket daemon qui reçoit les événements
acoustiques du Pi et les push dans une `queue.Queue`.

## Spécifications
- Thread daemon avec boucle asyncio interne
- Reconnect backoff : `WS_RECONNECT_BACKOFF` depuis `config.py` (1, 2, 5s)
- `queue.Queue(maxsize=50)` — `put_nowait`, drop si pleine (pas de blocage)
- Catch `InvalidProtocolMessage` → log warning sans crash
- Méthodes : `start()`, `stop()`, `get_event() -> AcousticEvent | None`
- `get_event()` non-bloquant (queue.get_nowait)
- Event de status connexion : `{"type": "status", "connected": bool}`

## Critères d'acceptation
- Importable sans erreur
- Se connecte à ws://localhost:8765 (ou host config)
- Reconnect automatique après coupure avec backoff
- Queue non-bloquante, drop events si pleine
- Ne crash pas sur message invalide
